## 5.1 Experimental Setup

All experiments were conducted on a machine running [FILL: OS + version],
with a [FILL: CPU model] processor and [FILL: RAM] GiB of RAM.
The simplification executable was compiled with `g++ -std=c++17 -O2`.
Wall-clock time was measured using `std::chrono::high_resolution_clock`
(external harness: `perf_counter` from Python's `time` module);
peak resident-set size (RSS) was sampled every 5 ms via the
`psutil` library and the maximum recorded value was reported.
Each measurement was repeated [FILL: reps] times and the **median** wall
time and the **maximum** observed peak RSS are reported to reduce noise.

### 5.1.1 Dataset Description

Four classes of synthetic polygons were generated by `benchmark/gen_datasets.py`:

| Class | Description | n range | Notes |
|-------|-------------|---------|-------|
| **large_n** | Smooth exterior-only polygon (Fourier perturbation of circle) | 50 – 12 800 | 9 sizes, target = ⌊n/2⌋ |
| **many_holes** | Fixed exterior (~300 v) + k small circular holes | k = 1–20 | Tests cross-ring intersection cost |
| **narrow_gap** | Concentric exterior + 1 hole, gap ∈ {5, 2, 1, 0.5, 0.1} | 200 | Stresses spatial-grid query density |
| **degenerate** | Comb (thin teeth), zigzag, elongated (50:1 aspect ratio), nearly-collinear | 20–2 000 | Near-zero displacements; fp robustness |

The large_n dataset is used for the primary scaling experiments (Sections 5.2
and 5.3); the remaining classes serve as stress tests (Section 5.4).

## 5.4 Areal Displacement vs. Target Vertex Count

> **Figure 3 here** — `fig3_displacement_vs_target.pdf`

The left panel plots total areal displacement against the absolute target
count; the right panel normalises by showing the percentage of vertices

Key observations:
- **Near-lossless simplification (< 20% removed):** areal displacement
  remains very small ([FILL] ≤ D ≤ [FILL]), indicating that the APSC
  greedy strategy successfully collapses only near-zero-cost candidates first.
- **Moderate simplification (20 – 60% removed):** displacement grows
  roughly linearly; the knee of the curve occurs at approximately
  [FILL]% removal ([FILL] target vertices), suggesting that below this
  threshold the algorithm is forced to collapse geometrically significant
  segments.
- **Aggressive simplification (> 60% removed):** displacement rises
  steeply as the algorithm exhausts low-cost candidates and must accept
  higher-displacement collapses.
- **Area preservation:** the signed area of the output polygon equals
  that of the input to within floating-point precision in all tested
  cases, confirming correctness of the area-preserving Steiner-point
  construction (§ 3.2).
